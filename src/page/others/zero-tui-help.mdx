## 项目简介

![zero-tui](/assets/images/zero/zero-tui.png)

**zero-tui** 是一款运行在终端里的 **TUI**（文本用户界面）工具，用来把分散在各 AI CLI / Agent 里的 **skills** 集中管理起来。你可以在里面完成这些事情：

- 安装与导入 skill
- 浏览本地 skill 列表与详情
- 维护各家 CLI 的 skills 目录映射
- 把全局 skill 仓通过软链同步回其它工具

它并不替代 Claude、Cursor、Codex 之类的客户端，而是给你一个**纯键盘操作、统一入口、路径清晰**的管理层。

如果你已经习惯使用 **skills.sh**、多套 skills 目录、以及各处软链同步，zero-tui 做的就是把其中**重复、易忘、路径容易混乱**的部分，统一收敛到 **`~/.zero-tui`** 这一处用户目录里。

## 安装方式

```bash
npm install --global zero-tui
```

安装完成后，在终端执行 **`z`** 即可启动。若你使用的是 **pnpm** 或 **yarn**，请改用对应的全局安装命令，并确保全局 `node_modules/.bin` 已加入 `PATH`。

## 快速理解它的工作方式

可以先用一句话理解 zero-tui：

> **把 `~/.zero-tui/skills` 当成唯一真相源，再把它同步回各个 AI 工具。**

它的典型关系如下：

| 角色     | 位置                                          | 作用                                      |
| -------- | --------------------------------------------- | ----------------------------------------- |
| 全局仓   | `~/.zero-tui/skills`                          | 统一存放你真正安装下来的 skills           |
| 配置文件 | `~/.zero-tui/config.json`                     | 保存 `linkTargetDirs` 等用户配置          |
| 目标目录 | 例如 Claude / Codex / Cursor 的 skills 根目录 | 由 `/skill update` 铺设软链，给各工具消费 |

## 数据放在哪里

| 路径                      | 含义                                                                      |
| ------------------------- | ------------------------------------------------------------------------- |
| `~/.zero-tui/skills`      | 全局 skill 安装目录（add / install / import / init 的主要落点）           |
| `~/.zero-tui/config.json` | 用户配置；其中 `linkTargetDirs` 记录各三方 CLI 的 skills **根目录**覆盖项 |

若你之前使用过旧目录下的配置，需要自行迁移到 `~/.zero-tui`（例如将原目录改名或合并），应用不会自动搬移。

## Slash 命令一览

下面这部分按“命令 / 用途 / 常见参数 / 备注”整理，方便查阅。具体行为仍以终端内 **`/help`** 为准。

### Skill 相关

| 命令                                         | 用途                              | 常见参数                       | 备注                                               |
| -------------------------------------------- | --------------------------------- | ------------------------------ | -------------------------------------------------- |
| `/skill add <name \| local-path \| git-url>` | 安装或接入一个 skill              | skill 名称、本地目录、git 地址 | 既支持从 **skills.sh** 下载，也支持本地 / git 来源 |
| `/skill import <local-dir>`                  | 从本机目录导入 skill 到 zero      | 本地目录                       | 适合把单个 skill 包或一个批量目录收进全局仓        |
| `/skill init`                                | 从其它 CLI 的 skills 目录回流导入 | 无                             | 常用于接手旧环境，或把各处分散的 skill 收口        |
| `/skill remove [name]`                       | 删除一个已安装的本地 skill        | skill 名称                     | 删除的是 `~/.zero-tui/skills` 中的对应内容         |
| `/skill path`                                | 查看全局 skill 仓路径             | 无                             | 可顺手在系统资源管理器里打开                       |
| `/skill target add [name] [absolute-dir]`    | 配置一个目标 skills 根目录        | 名称、绝对路径                 | 会写入 `config.json` 的 `linkTargetDirs`           |
| `/skill targets`                             | 查看所有目标路径与解析结果        | 无                             | 用来确认默认路径和自定义覆盖是否正确               |
| `/skill find <keyword>`                      | 搜索可安装 skill                  | 关键词                         | 基于 **skills.sh / skills CLI** 搜索               |
| `/skill list`                                | 查看本地已安装 skill 列表         | 无                             | 通常配合详情查看一起使用                           |
| `/skill info [name]`                         | 查看单个 skill 的详细信息         | skill 名称                     | 适合确认来源、描述和安装状态                       |
| `/skill update [target \| all]`              | 把全局 skill 仓同步为软链         | 目标名或 `all`                 | 真正执行“铺链接”的命令                             |

### 会话与其它

| 命令     | 用途                      | 常见参数 | 备注                    |
| -------- | ------------------------- | -------- | ----------------------- |
| `/help`  | 查看命令分组与快捷说明    | 无       | 第一次用时最值得先看    |
| `/clear` | 清空当前时间线 / 会话历史 | 无       | 不会影响本地 skill 数据 |
| `/exit`  | 退出 zero-tui             | 无       | 等同于结束当前应用会话  |

## 典型工作流建议

| 场景                        | 推荐做法                                                              |
| --------------------------- | --------------------------------------------------------------------- |
| 第一次使用                  | 启动 `z` 后先执行 `/help`，确认 `~/.zero-tui` 和 `config.json` 已生成 |
| 想安装一个新 skill          | 先 `/skill find` 搜关键词，再 `/skill add <name>` 安装                |
| 想接入本地或 git skill      | 直接使用 `/skill add <local-path \| git-url>`                         |
| 想统一旧环境的 skill        | 先 `/skill init` 回流，再 `/skill update` 铺回软链                    |
| 想校准路径是否正确          | 先看 `/skill targets`，确认后再跑 `/skill update`                     |
| 想自定义某个 CLI 的技能目录 | 使用 `/skill target add` 写入自定义绝对路径                           |

## 适合哪些人使用

- 同时使用多种 **AI CLI / Agent**，并希望 **skills 目录别各写一套**的人。
- 经常用 **skills.sh** 生态，想要终端里**可键盘操作**的安装与查阅体验的人。
- 需要维护 **`linkTargetDirs`**、在多台机器或多块磁盘间**对齐 skills 物理位置**的人。
- 喜欢 **Ink 系 TUI**、希望「历史区固定 + 底部输入不被顶掉」的人。

## 使用建议

如果你是第一次打开 zero-tui，建议按下面顺序**摸一遍**，会比死记命令更快：

1. 先 **`/help`**，读一眼配置路径和全局 skill 仓位置。
2. 再 **`/skill list`** 和 **`/skill path`**，确认本机 `skills` 目录结构。
3. 试一次 **`/skill targets`**，理解默认与 config 覆盖后的实际路径。
4. 最后在需要时跑 **`/skill update`**，感受软链同步过程。

zero-tui 最适合的使用方式，不是一次背全参数，而是把 **`~/.zero-tui` 当成唯一真相源**，需要时再用 **`/skill update`** 把链接铺回各个 CLI。
